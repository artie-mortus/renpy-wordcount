# Architectural Decisions

| Topic | Status | Decision / question |
|---|---|---|
| Unicode word counting | Decided (Steps 2.1/2.3) | Use a hand-written UTF-8 scanner backed by a generated, vendored table of Unicode general-category ranges for letters (`L*`) and numbers (`N*`). The checked-in table is Unicode 17.0, generated with Node 24.16.0 property escapes. Accept `'`, `’`, or `-` internally only when followed by another letter/number run, exactly matching `parser.js:121`: `/[\p{L}\p{N}]+(?:['’\-][\p{L}\p{N}]+)*/gu`. Invalid UTF-8 bytes are separators. This keeps `say_count_core` dependency-free and avoids ICU's binary/runtime cost; tests cover ASCII, accented, non-Latin, numeric, apostrophe, hyphen, and non-word emoji cases. |
| Settings location | Decided (Step 1.4) | Settings stored at `~/.local/share/say-count/settings.json` via `app::Settings` with atomic writes. wx 3.2 ignores `FileLayout_XDG` for `GetUserDataDir`, so the XDG path is resolved manually. `say_count_core` stays wx-free. |
| Parser fixture deferrals | Decided (Step 2.6C) | Defer `route analysis records inline menu and conditional branches` to Phase 7 Step 7.1 and `branch totals report dialogue inside menu choices and conditions` to Phase 7 Step 7.2. Defer both Ren'Py symbol collection/rename fixtures to Step 5.7. Step 2.6 does not implement route analysis or symbol rename. |
| Warning parity output | Decided (Step 2.6C) | Extend the JavaScript/native parity JSON with an ordered `warnings` array containing `type`, `file`, `lineNumber`, and `message`, while preserving every existing key. This makes parser and project-lint warning behavior part of byte-identical parity coverage. |
| Statistics targets | Decided (Step 3.4) | Keep word/line goals in the wx UI layer and persist them in `targets.ini` next to `settings.json` in the manually resolved XDG data directory (`wxStandardPaths::GetUserDataDir` returns legacy `~/.say-count`, same trap as Step 1.4; `MainFrame` injects the path). Project goals are scalar; character and scene goals are keyed by displayed aggregate name. Non-positive or invalid values are unset, without adding wx or target state to core or parity JSON. |
| Statistics export encoding | Decided (Step 3.6) | Generate CSV, JSON, and standalone HTML as UTF-8 in the wx-free core, write through wx's UTF-8 converter, quote every CSV field, JSON-escape control characters, and HTML-escape user-controlled labels/title. HTML declares UTF-8 explicitly. |
| Version comparison semantics | Decided (Step 3.7) | Match the JavaScript version diff by comparing parsed word totals, not textual tokens: net is current minus old; added/removed are the positive/negative sides of that net. Per-speaker rows union both versions and sort by absolute delta descending. The pasted/loaded baseline survives live current-document analysis rebuilds. |
| Outline scope and navigation | Decided (Step 4.1) | Build the outline in wx-free core code from the active document and its parser analysis. Qualify local labels against their current global label; resolved jumps/calls navigate to definitions, while unresolved references are red and navigate to their source line. Use double-click consistently with the counted-lines browser. Multi-file outline resolution waits for the project model in Step 5.1. |
| Basic autocomplete contexts | Decided (Step 4.2) | Keep context detection and completion construction in wx-free core code. Offer labels only after a bare `jump`/`call` target and aliases/snippets only when the line otherwise contains one identifier; never suggest inside strings or assignments. Let Scintilla own popup keyboard behavior, then replace the accepted display text with snippet/trailing-space content and select the declared placeholder range. Project-wide symbols wait for Step 5.1/4.3. |
| Asset autocomplete index | Decided (Step 4.3) | Build one wx-free completion index from every open editor document and replace it after tabs or text change. Collect declared image names, audio definitions and used quoted paths, screens, and variables (`default`, non-character `define`, and `$` assignments); also promote aliases and labels to project-wide completion. Keep filesystem asset scanning deferred until folder/project connectivity exists. |
| Find/replace semantics | Decided (Step 4.4) | Keep literal/regex matching and replacement in wx-free core code, with ASCII Ren'Py identifier boundaries for whole-word mode, non-overlapping matches, literal plain-text replacement, and ECMAScript-style `$1` capture expansion for regex replacement. Invalid regexes are reported without changing text. Use one Scintilla indicator for current-file highlight-all (capped at 2,000 rendered highlights) and group each replace operation into one undo action. |
| Cross-file find/replace safety | Decided (Step 4.5) | Search every open editor document in tab order and retain file index, byte range, one-based line/column, and trimmed source-line preview in the wx-free result model. Show results in a sortable docked list. Before project replacement, show per-file match counts, let the user select affected files, and require a second confirmation; compute all changes before applying them and give each changed tab one undo action. Connected-folder coverage will arrive automatically when folder sync populates the open-document model. |
