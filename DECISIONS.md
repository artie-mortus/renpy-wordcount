# Architectural Decisions

| Topic | Status | Decision / question |
|---|---|---|
| Unicode word counting | Decided (Steps 2.1/2.3) | Use a hand-written UTF-8 scanner backed by a generated, vendored table of Unicode general-category ranges for letters (`L*`) and numbers (`N*`). The checked-in table is Unicode 17.0, generated with Node 24.16.0 property escapes. Accept `'`, `’`, or `-` internally only when followed by another letter/number run, exactly matching `parser.js:121`: `/[\p{L}\p{N}]+(?:['’\-][\p{L}\p{N}]+)*/gu`. Invalid UTF-8 bytes are separators. This keeps `say_count_core` dependency-free and avoids ICU's binary/runtime cost; tests cover ASCII, accented, non-Latin, numeric, apostrophe, hyphen, and non-word emoji cases. |
| Settings location | Decided (Step 1.4) | Settings stored at `~/.local/share/say-count/settings.json` via `app::Settings` with atomic writes. wx 3.2 ignores `FileLayout_XDG` for `GetUserDataDir`, so the XDG path is resolved manually. `say_count_core` stays wx-free. |
| Parser fixture deferrals | Decided (Step 2.6C) | Defer `route analysis records inline menu and conditional branches` to Phase 7 Step 7.1 and `branch totals report dialogue inside menu choices and conditions` to Phase 7 Step 7.2. Defer both Ren'Py symbol collection/rename fixtures to Step 5.7. Step 2.6 does not implement route analysis or symbol rename. |
| Warning parity output | Decided (Step 2.6C) | Extend the JavaScript/native parity JSON with an ordered `warnings` array containing `type`, `file`, `lineNumber`, and `message`, while preserving every existing key. This makes parser and project-lint warning behavior part of byte-identical parity coverage. |
| Statistics targets | Decided (Step 3.4) | Keep word/line goals in the wx UI layer and persist them in `targets.ini` next to `settings.json` in the manually resolved XDG data directory (`wxStandardPaths::GetUserDataDir` returns legacy `~/.say-count`, same trap as Step 1.4; `MainFrame` injects the path). Project goals are scalar; character and scene goals are keyed by displayed aggregate name. Non-positive or invalid values are unset, without adding wx or target state to core or parity JSON. |
| Statistics export encoding | Decided (Step 3.6) | Generate CSV, JSON, and standalone HTML as UTF-8 in the wx-free core, write through wx's UTF-8 converter, quote every CSV field, JSON-escape control characters, and HTML-escape user-controlled labels/title. HTML declares UTF-8 explicitly. |
| Version comparison semantics | Decided (Step 3.7) | Match the JavaScript version diff by comparing parsed word totals, not textual tokens: net is current minus old; added/removed are the positive/negative sides of that net. Per-speaker rows union both versions and sort by absolute delta descending. The pasted/loaded baseline survives live current-document analysis rebuilds. |
| Outline scope and navigation | Decided (Step 4.1) | Build the outline in wx-free core code from the active document and its parser analysis. Qualify local labels against their current global label; resolved jumps/calls navigate to definitions, while unresolved references are red and navigate to their source line. Use double-click consistently with the counted-lines browser. Multi-file outline resolution waits for the project model in Step 5.1. |
| Basic autocomplete contexts | Decided (Step 4.2) | Keep context detection and completion construction in wx-free core code. Offer labels only after a bare `jump`/`call` target and aliases/snippets only when the line otherwise contains one identifier; never suggest inside strings or assignments. Let Scintilla own popup keyboard behavior, then replace the accepted display text with snippet/trailing-space content and select the declared placeholder range. Project-wide symbols wait for Step 5.1/4.3. |
| Asset autocomplete index | Decided (Step 4.3) | Build one wx-free completion index from every open editor document and replace it after tabs or text change. Collect declared image names, audio definitions and used quoted paths, screens, and variables (`default`, non-character `define`, and `$` assignments); also promote aliases and labels to project-wide completion. Keep filesystem asset scanning deferred until folder/project connectivity exists. |
| Find/replace semantics | Decided (Step 4.4) | Keep literal/regex matching and replacement in wx-free core code, with ASCII Ren'Py identifier boundaries for whole-word mode, non-overlapping matches, literal plain-text replacement, and ECMAScript-style `$1` capture expansion for regex replacement. Invalid regexes are reported without changing text. Use one Scintilla indicator for current-file highlight-all (capped at 2,000 rendered highlights) and group each replace operation into one undo action. |
| Cross-file find/replace safety | Decided (Step 4.5) | Search every open editor document in tab order and retain file index, byte range, one-based line/column, and trimmed source-line preview in the wx-free result model. Show results in a sortable docked list. Before project replacement, show per-file match counts, let the user select affected files, and require a second confirmation; compute all changes before applying them and give each changed tab one undo action. Connected-folder coverage will arrive automatically when folder sync populates the open-document model. |
| Diagnostics engine boundary | Decided (Step 4.6) | Keep native editor diagnostics in a dedicated wx-free model rather than adding native-only rules to `ScriptAnalysis.warnings`, preserving byte-identical parser parity JSON. Map the existing project-aware label, quote, speaker, length, and syntax warnings to severity and byte ranges, then add tab/non-four-space indentation notices. Suppress generated Ren'Py support files consistently with parser lint and rebuild from current open documents so fixed issues disappear. |
| Diagnostics presentation | Decided (Step 4.7) | Render error, warning, and notice diagnostics with separate Scintilla squiggle indicators and colored circle markers in a dedicated symbol margin. Clear all diagnostic indicators/markers before applying each debounced project snapshot. Show every message on line dwell and in a sortable docked panel; panel activation switches tabs and selects the diagnostic byte range. |
| Editing command transactions | Decided (Step 4.8) | Keep selection-aware comment, move, duplicate, pair, and label-navigation transformations in wx-free core code. Apply text-changing commands to Scintilla as one undo action and preserve byte selections/indentation exactly. Auto-close `"`, `(`, and `[`, wrap selected text, type over existing closers, and let escaped quotes fall through to normal insertion. Use native menu accelerators for Ctrl+G/Ctrl+K and editor key handling for context-sensitive commands. |
| Focus-mode layout restoration | Decided (Step 4.9) | Serialize the complete wxAUI perspective immediately before entering focus mode, hide every managed pane except the editor, and reload that exact perspective when leaving. Keep the live dialogue-word pill as an unmanaged frame overlay so it is never serialized into or able to alter the saved docking layout. Refit/reposition the pill after count and frame-size changes. |
| Project folder identity | Decided (Step 5.1) | Represent a connected project independently from open tabs with a canonical project root, scripts root, and sorted recursive `.rpy` file list. Selecting a project containing `game/` watches/opens that directory; selecting `game/` records its parent as the project root. Open every discovered script while removing only untouched blank placeholders. Persist up to eight deduplicated recent canonical roots in the existing atomic settings file. |
| External change safety | Decided (Step 5.2) | Watch the connected scripts tree recursively with `wxFileSystemWatcher` and classify normalized disk content against the editor buffer and Scintilla save point. Ignore identical revisions, reload only clean divergent tabs while preserving selection/scroll, and retain dirty divergent buffers untouched as pending conflicts for Step 5.3. Treat create/rename events like modifications to support atomic-save editors; preserve open tabs on deletion and refresh project discovery for tree changes. |
| Conflict resolution integrity | Decided (Step 5.3) | Retain complete local and disk strings for every dirty external conflict and never synthesize a merged buffer. Build an aligned side-by-side line model from common prefix/suffix context and highlight the changed middle region. Keep-local leaves Scintilla dirty, use-disk replaces the buffer and sets its save point, and save-elsewhere must successfully write the local version to a different path before use-disk proceeds. If the disk revision changes during modal review, discard the stale decision and reopen with fresh content. |
| Snapshot storage | Decided (Step 5.4) | Store snapshots as versioned binary files under the existing user-data directory, preserving UTF-8 file names and exact script bytes with length-prefixed fields. Write atomically through a temporary file and rename; assign strictly monotonic millisecond IDs, suppress consecutive duplicate project states, and retain the newest 50 snapshots. Record label, automatic/manual origin, project root, file count, and word count so Step 5.5 can present restore and comparison choices without reopening every file. |
| Snapshot restore safety | Decided (Step 5.5) | Preview a selected snapshot against the complete current tab set by file name, content, and word count, classifying added, removed, changed, and unchanged files. Require explicit confirmation, snapshot the outgoing editor state first, then replace the complete tab set without writing to disk. Preserve paths for matching open files and leave restored differences dirty so saving remains an explicit user action. |
| Project bundle compatibility | Decided (Step 5.6) | Implement the browser app's version-1 `say-count-project` JSON boundary in wx-free core code. Validate format/version, non-empty named file content pairs, settings, and target maps before mutating UI state; normalize tabs through the existing editor load path. Import snapshots the outgoing project and restores file order/active index, theme, menu-choice counting, and all targets. Export emits the same files/settings/targets schema and retains semantically valid unknown top-level fields from imported bundles. |
| Symbol rename safety | Decided (Step 5.7) | Restrict project-wide rename to syntactically supported Ren'Py contexts: `Character` alias declarations, leading dialogue aliases, label declarations, and static `jump`/`call` targets (including calls with arguments). Validate alias/label syntax, reject replacement declaration collisions, and preserve exact line endings while ignoring comments, quoted prose, Python, similar substrings, and expression-based flow. Preview every affected line and require a successful snapshot before applying changed files as undoable dirty editor transactions. |
| Ren'Py SDK discovery | Decided (Step 6.1) | Persist an explicit SDK executable in the existing atomic editor settings, then detect in priority order: valid configured path/directory, `RENPY_EXECUTABLE`, `PATH`, and conventional `~/renpy-*-sdk` or `~/Downloads/renpy-*-sdk` folders. Require a regular executable file, infer versions from SDK folder names, and parse launcher `--version` output when configured interactively. Keep discovery and version parsing wx-free and expose current detection in a dedicated Ren'Py menu. |
| Ren'Py process ownership | Decided (Step 6.2) | Launch the detected SDK asynchronously with an argv array (`executable`, project root, operation arguments), never through a shell string. Give `MainFrame` sole ownership of one redirected `wxProcess`, reject overlaps, poll and merge stdout/stderr into a bounded docked view, and append the complete stream to `renpy-launch.log` under user data. Stop the process tree with `SIGTERM`, report exit status, and detach/terminate an owned child during application teardown. |
| Warp and Director launch semantics | Decided (Step 6.3) | Before warp or Interactive Director, save every modified disk-backed tab and abort on unsaved tabs or write failure. Compute warp targets as forward-slash paths relative to the connected scripts/game root plus the one-based caret line, then invoke `--warp path:line`; invoke Director with the SDK `director` command. Allow unknown custom-launcher versions, but block known versions older than the 6.99 generation where these developer workflows are unsupported. Reuse the single captured process/log path so every command and failure remains visible. |
