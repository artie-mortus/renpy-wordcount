CHAT FORMAT · SOCIAL-MEDIA SCENES

Write messenger scenes as prose. Play them as a chat app.

Say Count converts prose or ordinary dialogue into the chat_program framework's Ren'Py syntax, which renders scenes as a phone/messenger app inside your game. The reverse conversion turns supported chat scenes back into editable dialogue. No coding is needed for the normal workflow.

Quick start

1. Write the scene as plain text, one line per message: Robo: yo you seeing this?? Use Me: for the player, and stage directions like [Robo is typing] or Choice: blocks for chat features (see below).
2. Select it and choose Edit → Turn Writing Into a Chat Scene…
3. Pick a look (Discord or Kik) and check the What players will see preview tab. The exact code that will be inserted is on the Script code tab.
4. Click Use this. If the chat app is not installed in your project yet, Say Count offers to install it for you.
5. Changed your mind? Undo puts your original text back.

Narrator handling and character short codes live under Advanced options in the conversion window; the defaults work without touching them.

The three commands

Edit → Turn Writing Into a Chat Scene… converts the selection, or the whole active tab, into chat messages. Pick a look, check the preview, and click Use this — one undoable replacement.

Edit → Turn Chat Scene Back Into Dialogue… converts a chat scene back to plain writing (Name: text) or Ren'Py say statements. Chat-only details that cannot be kept are listed and must be acknowledged first.

Edit → Install/Update Chat App Files… copies the pinned chat_program runtime into game/vendor/chat_program. Say Count also offers this automatically after your first conversion. Locally modified files are never overwritten.

All three are also in the command palette (Ctrl+Shift+P).

Setup, once per project

Connect your project folder and convert a scene — Say Count will offer to install the chat app files for you. (You can also run Install/Update Chat App Files yourself at any time.) The installer adds the framework, license, and a project-owned game/say_count_chat_config.rpy that is yours to edit and is never overwritten afterward.

Writing a chat scene

Type the scene as prose, select it, and convert:

WRITE
Robo: yo you seeing this??
Me: seeing what		 BECOMES
r "yo you seeing this??"
m "seeing what"

Character definitions are generated when missing:

default r = ChatCharacter("Robo", icon="images/robo.png")
default m = ChatCharacter("[player_name]", is_player=True)

Generated short codes use lowercase initials: Leon becomes l and Captain Vale becomes cv. If an abbreviation is already in use, the next character receives _2, then _3, and so on.

Stage directions: chat features without code

Write square-bracket stage directions on their own line, like a screenplay. The converter turns them into the right code for you:

[in #announcements] or [to Eileen] — move the scene to another room or conversation thread.
[Robo is typing] or [Robo and Mel are typing] — show a typing indicator on the next message.
[fast], [fast 0.25], [normal speed] — speed up or restore message pacing.
[I am Eileen] — mark who the player is, so their bubbles align right in the messenger view. A speaker written as Me: becomes the player automatically.

A direction the converter does not understand stays in the scene as narration and is listed as a warning, so nothing disappears silently.

Player replies (choices)

Write a Choice: line, then one - line per reply. Indent lines under a reply to show what happens after the player picks it:

Robo: you coming?
Choice:
- Sounds good
    Robo: nice
- No way

Converting a chat scene back to writing produces these same stage directions and choice blocks, so a scene can go back and forth without losing its chat behavior.

Raw arguments (optional)

Writers who prefer code can still pass chat_program arguments directly; both forms can be mixed:

robo "check #announcements" (c="#announcements", ot=[me], fastmode=0.1)

c= switches channel, ot= shows other characters typing, and fastmode= compresses message pacing. Menus support auto_send=False for choices the player must confirm.

App styles: Discord or Kik

The conversion dialog asks which app the scene should look like. Both styles use identical syntax and the same message engine; only the installed screen changes.

Discord-style channels shows a server sidebar with #channel tabs and flat, avatar-led messages.

Kik-style messenger shows a portrait phone with chat bubbles: the player's messages align right, contacts align left with avatar and name, and a thread list replaces the channel sidebar. Name channels after contacts (c="Eileen") so each reads as a direct-message thread.

The Make a complete scene checkbox (on by default) emits a self-contained scene that opens the chosen app and returns to normal VN dialogue at the end. Untick it when converting a fragment for a scene that already has say_count_chat_begin/end bridge calls. A project-wide default lives in say_count_chat_config.rpy (say_count_chat_skin).

Entering and leaving the chat app

Mix chat and normal VN scenes in one script with the bundled bridge labels:

label chapter_one:
    e "Normal VN dialogue."

    call say_count_chat_begin("#general")
    robo "yo you seeing this??"
    me "seeing what"
    call say_count_chat_end

    scene bg bedroom
    e "Back to the regular visual novel."

say_count_chat_begin clears earlier messages by default; pass clear=False to keep history, as when the player reopens their phone. say_count_chat_end hides the chat screen and restores the normal dialogue window.

What the reverse conversion loses

Regular dialogue has no place for channels, other-typer lists, fastmode, player ownership, icons, name colors, or auto_send. The preview lists each loss; unrecognized code is preserved unchanged rather than guessed at.

Notes

  • Say Count never edits screens.rpy. Copy the upstream choice screen if you want chat-style auto-send replies.
  • Keep project settings in say_count_chat_config.rpy; avoid editing the vendored runtime so upgrades stay automatic.
  • See game/vendor/chat_program/INTEGRATION.md in your project for the full integration reference.

If a conversion is not what you expected, use Undo.
